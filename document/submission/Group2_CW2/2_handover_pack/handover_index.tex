\section{What is Byte Me?}

Byte Me is a web application where food sellers list surplus bundles at discounted prices and charitable organisations reserve them for collection. The platform tracks environmental impact (meals rescued, CO2e saved), provides demand forecasting for sellers, and uses gamification (streaks, badges) to keep organisations engaged.

\subsection{For Sellers}

\begin{itemize}
  \item Create and manage surplus food bundles with pickup windows
  \item View analytics dashboards: sell-through rates, pricing effectiveness, popular categories
  \item Get demand forecasts with model comparison and quantity recommendations
  \item Handle issue reports from organisations
\end{itemize}

\subsection{For Organisations}

\begin{itemize}
  \item Browse and reserve surplus bundles from local sellers
  \item Collect orders using claim codes with pickup time reminders
  \item Track impact stats: meals rescued, CO2e saved, streaks
  \item Earn badges through consistent food rescue activity
  \item Report quality or availability issues
\end{itemize}

\section{Getting Started}

\subsection{Prerequisites}

\begin{itemize}
  \item Java 17
  \item Maven
  \item Node.js 20+
  \item PostgreSQL
\end{itemize}

\subsection{1. Clone the repo}

\begin{lstlisting}
git clone https://github.com/MarcosAsh/Byte_Me.git
cd Byte_Me
\end{lstlisting}

\subsection{2. Set up the database}

\begin{lstlisting}
CREATE DATABASE byte_me;
\end{lstlisting}

\subsection{3. Configure environment variables}

Create \texttt{backend/.env} with:

{\small
\begin{longtable}{l l l}
\toprule
Variable & Description & Example \\
\midrule
\texttt{SERVER\_PORT} & Backend server port & \texttt{8080} \\
\texttt{DB\_URL} & PostgreSQL JDBC URL & \texttt{jdbc:postgresql://localhost:5432/byte\_me} \\
\texttt{DB\_USERNAME} & Database username & \texttt{postgres} \\
\texttt{DB\_PASSWORD} & Database password & \texttt{postgres} \\
\texttt{JWT\_SECRET} & Signing key for JWTs (32+ bytes) & \texttt{change-me} \\
\texttt{JWT\_EXPIRATION} & Token expiry in ms & \texttt{86400000} \\
\bottomrule
\end{longtable}
}

\subsection{4. Start the backend}

\begin{lstlisting}
cd backend
mvn spring-boot:run
\end{lstlisting}

Runs on \texttt{http://localhost:8080}.

\subsection{5. Start the frontend}

\begin{lstlisting}
cd frontend
npm install
npm run dev
\end{lstlisting}

Runs on \texttt{http://localhost:3000}.

\subsection{Optional: automated setup scripts}

\begin{lstlisting}
# macOS
cd scripts && chmod +x setup-mac.sh && ./setup-mac.sh

# Windows (PowerShell as Administrator)
cd scripts; .\setup-windows.ps1
\end{lstlisting}

\section{Running Tests}

\subsection{Backend Tests}

Run all backend tests:

\begin{lstlisting}
cd backend
mvn clean test
\end{lstlisting}

Maven will output the results to the terminal. Detailed reports can be found in \texttt{target/surefire-reports/}.

Run a specific test class:

\begin{lstlisting}
cd backend
mvn test -Dtest=IssueControllerTest
\end{lstlisting}

Run a specific test method:

\begin{lstlisting}
cd backend
mvn test -Dtest=BundleControllerTest#testGetById_Success
\end{lstlisting}

\subsection{Frontend Tests}

The frontend uses Vitest with React Testing Library (191 tests across 13 test files).

Install test dependencies (first time only):

\begin{lstlisting}
cd frontend
npm install -D vitest @vitejs/plugin-react \
  @testing-library/react \
  @testing-library/user-event \
  @testing-library/jest-dom jsdom
\end{lstlisting}

Run all frontend tests:

\begin{lstlisting}
cd frontend
npx vitest run
\end{lstlisting}

Run a specific test file:

\begin{lstlisting}
npx vitest run login
\end{lstlisting}
